# Change Management

## Purpose
Ensure changes to AssetControl are reviewed, approved, tested, and traceable.

## Process
1. Change request created with impact assessment.
2. Risk and security review for sensitive changes.
3. Implementation in feature branch.
4. Peer review and approval.
5. Deploy with rollback plan.
6. Post-change verification and record update.

## Emergency Changes
- Allowed only when service is at risk.
- Must be documented retroactively within 24 hours.

## Records
- Store change records in the issue tracker or release notes.
