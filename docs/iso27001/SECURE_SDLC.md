# Secure SDLC

## Objectives
- Build security into design, development, testing, and release.
- Reduce vulnerabilities and ensure traceable changes.

## Requirements
- Threat modeling for new features.
- Code review required for all merges.
- Dependency scanning on every release.
- Unit and integration tests for auth, permissions, and audit logs.

## Checkpoints
- Design review: security requirements identified.
- Development: secure coding practices, input validation, error handling.
- Testing: automated tests and vulnerability scans.
- Release: change approval and rollback plan.

## Evidence
- PR reviews and CI reports.
- Test results (vitest).
- Dependency audit outputs.
